# Year 1

> Goal: Master Python, C, C++ and prepare for Assembly — while actively bug hunting and building hacking tools.



## ✅ Month 1–2: Python Core (Daily Practice + Real Use)
Goal: Be fluent enough to automate recon, write PoCs, and scripts.

### Topics
- Variables, control flow, data types
- Functions, modules, classes
- File handling, subprocess, JSON
- Requests, basic CLI tools

### Projects
- HTTP scanner wrapper (httpx, nmap)
- Script for subdomain + IP resolution
- Token bruteforce automation



## ✅ Month 3: Python for Hackers
Goal: Use Python to write hacking tools and small exploits.

### Topics
- Socket programming
- HTTP requests, proxies
- Buffer overflow PoCs
- Parsing HTML & JSON responses

### Projects
- Subdomain scanner
- Basic port scanner
- Automated IDOR tester



## ✅ Month 4–6: Learn C (Systems & Malware Basics)
Goal: Understand low-level programming, memory, and binary structure.

### Topics
- Pointers, memory allocation
- Structs, arrays, string manipulation
- Shellcode basics, raw sockets
- Compilation, linking, debugging

### Projects
- Keylogger (local, CLI)
- File dropper
- Basic reverse shell



## ✅ Month 7–8: Learn C++ (Object-Oriented Malware Frameworks)
Goal: Understand object-oriented malware logic & advanced tooling.

### Topics
- Classes, inheritance
- Templates, STL
- C++ malware structure
- Stealth techniques

### Projects
- Modular RAT (console-based)
- Encrypted payload loader
- Rewrite C tools using C++



## ✅ Month 9–10: Projects + Bug Bounty Scaling
Goal: Combine knowledge into custom tools and active bug hunting.

### Activities
- maxRecon full-featured CLI version
- Combine all scripts/tools into one toolkit
- Submit bugs using my own PoCs
- Automate recon + testing pipeline



## ✅ Month 11–12: Malware Simulation + Reverse Engineering Prep
Goal: Simulate malware behavior and prepare for Assembly.

### Topics
- Debugging: GDB, x64dbg, OllyDbg
- Disassembling: Ghidra, IDA Free
- Malware anatomy: PE headers, syscalls
- Basic obfuscation, AV evasion

### Activities
- Analyze real malware in VM
- Write simple dropper/loader
- Reimplement malware logic in my own way



# Notes
